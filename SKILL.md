# 抖音销售利润计算器

## 功能说明
根据抖音电商销售报表，自动匹配成本数据，计算每笔订单的利润，并生成包含3个工作表的Excel利润分析报告。

## 使用场景
- 用户发送抖音销售报表文件（Excel/CSV）
- 用户说"抖音利润"、"利润计算"、"销售利润"等关键词
- 用户要求分析抖音销售数据

## 文件位置
- 成本表：`D:\01A工作\抖音订单利润计算\cost_table.csv`（178个SKU的成本数据，含百家布全系列）
- 销售报表目录：`D:\01A工作\抖音订单利润计算\每日订单\`
- 输出报表目录：`D:\01A工作\抖音订单利润计算\每日订单\`
- 计算脚本：`scripts/calculate_profit.py`

## 执行流程（严格执行，不要用模型读取文件）

当用户发送销售报表文件或触发关键词时，**必须**按以下步骤执行：

### 第1步：定位文件
- 从 `~/.hermes/cache/documents/` 查找最新接收的文件（按修改时间排序取最新）
- 不要用模型（pandas/read_file）读取销售报表内容，直接交给脚本处理

### 第2步：运行计算脚本（直接调用，不要读取文件内容）
```bash
python3 ~/.hermes/skills/finance/douyin-profit-calculator/scripts/calculate_profit.py --sales "<销售报表文件路径>" --output "D:/01A工作/抖音订单利润计算/每日订单/利润表_<时间戳>.xlsx"
```

### 第3步：发送报表给用户
- 将生成的Excel文件路径发送给用户
- 在消息中附带关键统计数据汇总（总利润、最高利润商品等）
- 文件永久保存在 `D:\01A工作\抖音订单利润计算\每日订单\`

### 第4步：清理销售报表
- 计算完成后，自动删除发送过来的销售报表源文件（cache目录下的临时文件）
- 命令：`rm "<销售报表文件路径>"`
- 利润报表已保存到永久目录，源文件不再需要

⚠️ **禁止事项：**
- 不要用 pandas read_csv/read_excel 读取销售报表
- 不要用 read_file 读取销售报表
- 不要在模型中解析报表内容
- 所有数据处理必须在 calculate_profit.py 脚本内完成
- 不要擅自修改此技能，除非用户明确要求
- **生成结果后直接发送利润统计数据摘要 + 文件路径即可，文件已自动保存到 `D:\01A工作\抖音订单利润计算\每日订单\`**

### 成本匹配失败处理
如果脚本输出中包含「未匹配成本」或有商品成本为0的情况，需要：
1. 将未匹配的商品名称列表发送给用户
2. 等待用户提供这些商品的成本信息（商品成本、物流成本、包装成本）
3. 用户补充后，更新 `D:\01A工作\抖音订单利润计算\cost_table.csv` 成本表

## 计算逻辑（严格遵循原始脚本）

### 数据清洗规则
排除以下订单：
- 订单状态为「待支付」或「已关闭」
- 商品数量 > 10
- 订单应付金额 > 350
- 订单应付金额 = 0（退款订单，平台补贴也不存在）

### 利润公式
```
单利润 = 商家收入金额 × 0.95 - 商品数量 × (商品成本 + 物流成本 + 包装成本)
```

说明：
- `商家收入金额 × 0.95` 是扣除5%平台服务费后的实际收入
- 成本按「商品成本 + 物流成本 + 包装成本」三项分别乘以数量

### 成本匹配逻辑
**第1步：精确匹配** — 按商品名完全匹配成本表（178条记录）

**第2步：模糊匹配** — 未匹配的商品按「产品类型+尺寸」规则匹配（详见下方「模糊匹配规则」章节）

### 输出报表结构
**Sheet1 - 利润总览**（按商品分组汇总）
| 列名 | 说明 |
|------|------|
| 选购商品 | 商品名称 |
| 总销量 | 该商品总销售数量 |
| 总商家收入 | 该商品商家收入金额合计 |
| 总商品成本 | 商品成本合计 |
| 总物流成本 | 物流成本合计 |
| 总包装成本 | 包装成本合计 |
| 总利润 | 利润合计 |

按总利润降序排列。

**Sheet2 - 有效订单**（通过清洗的订单明细）

**Sheet3 - 筛选订单**（被过滤掉的订单明细）

## 注意事项
- 成本表位于 `D:\01A工作\抖音订单利润计算\cost_table.csv`，用户可自行维护更新
- CSV成本表必须带UTF-8 BOM头，否则Excel打开会乱码（脚本用 `encoding='utf-8-sig'` 读取兼容BOM）
- 订单编号列需保留为文本格式（防止科学计数法）
- 如果商品在成本表中找不到匹配，会尝试模糊匹配（见下方规则），仍未匹配则成本为0
- 支持CSV和Excel格式的销售报表
- **不要擅自修改此技能**，除非用户明确要求
- **GitHub备份**：`https://github.com/iminddesign/douyin-profit-calculator`

## 模糊匹配规则（百家布产品）

精确匹配失败后，脚本自动按「产品类型 + 尺寸」模糊匹配。匹配前会标准化名称（去除cm、±5cm、空格等）。

**优先级从上到下，首条匹配即停止：**

| 产品类型 | 90*220 | 150*200 | 200*230 | 230*250 |
|----------|--------|---------|---------|---------|
| 夏凉被/被 | 13.5/2.5/0.5 | 32/4.5/0.5 | 39/4.5/0.5 | — |
| 三件套 | 25.3/2.5/0.5 | — | 45.8/3.6/0.5 | 54.8/4.5/0.5 |
| 四件套 | 38.3/2.5/0.5 | — | 84.8/3.6/0.5 | 93.8/4.5/0.5 |
| 单床盖 | 13.5/2.5/0.5 | — | 34/3.6/0.5 | 43/4.5/0.5 |
| 枕套 | — | — | — | 11.8/1.7/0.5 (48*74) |
| 垫子 | 13.5/2.5/0.5 | — | — | — |

格式：商品成本/物流成本/包装成本

**关键区分规则：**
- 三件套：必须包含"三件套"关键词，不会误匹配单件
- 四件套：必须包含"四件套"关键词
- 单床盖：排除包含"三件套""四件套"的商品
- 尺寸格式兼容：200cm*230cm、200*230、90cm*220cm、90*220 等均可匹配

**新增产品时：** 如果模糊规则无法覆盖，需同时更新 `cost_table.csv` 和 `scripts/calculate_profit.py` 中的 `fuzzy_rules` 列表。
